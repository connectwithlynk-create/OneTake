OneTake (working title)

Mobile app that collapses filming, reviewing, and editing of short-form video into one loop.

The creator records clips; the app returns an instant verdict on each take (dud / keep / perfect) and auto-tags it (talking vs b-roll). On Finish, the app auto-edits a publish-ready vertical video: bad takes and dead air removed, captions burned in, relevant b-roll cut in automatically. A separate prompt mode generates a video from the existing footage library on request. An Inspiration section lets the creator swipe-collect reels into named collections and later reuse a collection as the editing-style reference for a finished video.

Core bet (the moat): the talking-head auto-edit. Instant rating/tagging earns trust during the shoot; the one-tap auto-edit is the premium payoff. Everything else feeds or frames that output.

Decisions locked (2026-05-17)

PlatformExpo with prebuild (dev client / EAS Build). Not Expo Go. WedgeTalking-head auto-edit is the differentiator the product lives or dies on. 2. Problem

Creators spend far more time reviewing footage and editing than filming.

PainDetail
Review is post-hocSorting usable takes happens after the shoot, when context is lost.
Editing is repetitiveCut silences, remove fumbles, add captions, find and place b-roll, every time.
Style driftsConsistency across videos requires re-deriving the same editing choices by hand.

3. Goals and non-goals

Goals

  • Tell the creator within ~1s of stopping a take whether it is usable.
  • Eliminate manual cutting of silences, filler, and bad takes for talking-head projects.
  • Produce a publish-ready vertical video with captions and auto-placed b-roll in one tap.
  • Make collecting and reusing inspiration frictionless.

Non-goals (out of product scope)

  • Full manual timeline editor competing with CapCut/Premiere. Manual override is for corrections only.
  • Long-form / horizontal / multi-speaker production.
  • Music licensing marketplace.
  • Social feed or in-app distribution. OS share sheet is the publishing path.

4. Target users

PrimarySolo short-form creators (TikTok / Reels / Shorts) who film talking-head content with b-roll and edit on their phone. SecondaryFounders / marketers producing regular talking-head clips who want consistency without an editor. Persona traitsFilms in bursts, redoes lines multiple times, low patience for editing, judges quality by "would I post this".

7. User journeys

7.1 Talking-head project (primary)

Start Talking-Head project → record clip, stop → within ~1s: verdict chip + tag, both overridable → re-record lines as needed, duds de-emphasized → Finish → upload kept clips + manifest, backend transcribes/edits/renders → preview screen with the finished video, optionally re-render with a chosen inspiration style, export/share.

7.2 Prompt project

Start Prompt project, type prompt → backend semantically selects and orders matching library clips, assembles, captions, renders → same preview/export screen. Selects existing footage, never invents footage that does not exist.

7.3 Inspiration

Open Inspiration, import reels by URL or share sheet → swipe right into a chosen named collection, left to discard → collections are named → on any finished video, pick a collection so the re-render approximates that style.

9. Functional requirements

Capture

IDRequirement
FR-CAP-1Record multiple discrete clips within one project, in order.
FR-CAP-2Front/back camera, tap-to-focus, exposure lock, basic timer.
FR-CAP-3Each clip stored locally with metadata (project id, order index, duration, created at, orientation, resolution).
FR-CAP-4Vertical 1080x1920 capture target.

Instant rating

FR-RATE-1Verdict in {dud, keep, perfect} within ~1s of recording end (analysis of just-recorded file; not required to be live during recording).
FR-RATE-2Inputs: face presence/centering and eyes-open (talking), blur (Laplacian variance), audio level + clipping, clip length sanity, speech presence.
FR-RATE-3Verdict overridable; override stored.
FR-RATE-4Duds excluded from auto-edit by default but retained on device until deleted.

Auto-tagging

FR-TAG-1Auto-classify each clip talking vs b-roll (speech + talking-face => talking; else b-roll).
FR-TAG-2Tags overridable and stored.
FR-TAG-3Free-form / additional clip tags.v2

B-roll library

FR-BROLL-1Ship a bundled, pre-tagged b-roll set in the app.
FR-BROLL-2Each b-roll item has a text description used for matching.
FR-BROLL-3Creator-added b-roll; description auto-generated by a vision model, editable.v2

Auto-edit (talking-head)

FR-EDIT-1Concatenate kept clips in record order.
FR-EDIT-2Transcribe with word-level timestamps.
FR-EDIT-3Remove silences over a configurable threshold and filler words from a maintained list.
FR-EDIT-4Generate a caption track from word timings (highlight style).
FR-EDIT-5Match transcript spans to b-roll by embedding similarity; overlay matched b-roll as a cutaway while keeping the talking audio.
FR-EDIT-6Render vertical 1080x1920 H.264 MP4.
FR-EDIT-7Retake dedup: when a line is repeated, an LLM selects the best take and drops the rest.v2
FR-EDIT-8Manual corrections on the result: toggle a cut, remove a b-roll overlay, edit a caption; re-render applies them. Correction, not a timeline.

Prompt mode

FR-PROMPT-1Free-text prompt input on a Prompt project.
FR-PROMPT-2Semantic selection and ordering of library clips (transcript + tag + description embeddings).
FR-PROMPT-3Same caption + render pipeline as auto-edit.
FR-PROMPT-4Scope guard: selects/orders/edits existing footage; does not generate footage that does not exist.

Inspiration

FR-INSP-1Add reels/TikToks by URL or OS share sheet.
FR-INSP-2Swipe right to file into a chosen named collection, left to discard.
FR-INSP-3Create / rename / delete collections.
FR-INSP-4View a collection as a moodboard grid.
FR-INSP-5Style-match: derive measurable traits (mean shot length, caption density, music energy, hook pattern) and apply as render parameters for a re-render.v3

Library, projects, export

FR-LIB-1List projects with status (recording, processing, ready).
FR-LIB-2Per project: clips with verdict/tag, finished video(s).
FR-LIB-3Delete clips/projects; deleting reclaims local storage.
FR-EXP-1Export the finished MP4 via the OS share sheet and save to camera roll.

10. Non-functional requirements

IDRequirement
NFR-PERF-1Verdict latency ~1s after stop (P50), 2s (P95).
NFR-PERF-2Auto-edit round trip (upload + transcribe + edit + render) under 1.5x source footage duration for MVP-capped projects.
NFR-LIMIT-1MVP caps a talking-head project at ~2 minutes total footage to bound latency and cost.
NFR-OFFLINE-1Capture, rating, tagging, and library work fully offline. Auto-edit and prompt mode require connectivity.
NFR-PRIV-1Clips leave the device only on Finish. State clearly in UI; provide a way to delete server-side copies after render.
NFR-COST-1Track per-render cost (transcription + render compute) as a product KPI; it gates pricing.
NFR-PLAT-1iOS first. Codebase stays cross-platform; Android is a later hardening pass, not a rewrite.

11. System architecture

App captures, rates, tags, and emits a manifest. All editing and rendering is server-side (forced by the FFmpegKit retirement, section 5). Remotion chosen as renderer because the EDL maps directly to React component props and it is the same language as the app. Tradeoff accepted: render latency roughly proportional to video length, plus per-render compute cost (tracked via NFR-COST-1).

12. EDL contract (core data structure)

The app never edits video. It emits a manifest (clips + verdicts + tags). The backend turns manifest + transcript into an Edit Decision List. The renderer turns the EDL into a file. This isolation makes the moat testable before any UI exists.

EDL {
  output: { width, height, fps }
  tracks: [
    { type: "video",    segments: [ { clipId, sourceIn, sourceOut, timelineStart } ] }
    { type: "broll",    overlays: [ { brollId, timelineStart, timelineEnd } ] }
    { type: "captions", cues:     [ { text, start, end, words:[{w,start,end}] } ] }
  ]
  style?: { meanShotLen, captionStyle, musicEnergy }   // v3 from Inspiration
}

15. AI / ML components

ComponentApproach
On-device verdict/tagFace + eyes (MLKit via Vision Camera frame data), blur (Laplacian variance), audio RMS/clipping from file, speech presence. Heuristic scoring for MVP; learnable later from override data.
TranscriptionDeepgram or OpenAI Whisper API. Word-level timestamps required.
EmbeddingsTranscript spans and b-roll descriptions for matching; reused for prompt-mode semantic selection.
LLM (Claude, latest capable)v2 retake dedup; v3 prompt understanding. Transcript-level reasoning, not video.

17. MVP definition

In (end of Phase 4)

Talking-head record, instant rating, auto-tag, bundled b-roll, one-tap server auto-edit (cut + captions + b-roll), preview, export, iOS only.

Out

Prompt mode, style-matching, user b-roll, retake dedup, Android polish.
